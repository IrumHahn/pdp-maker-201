# RE:BRANDB Client Portal 화면 구조도 IA

업데이트 일자: 2026-03-11
기준 프로젝트: 빈파트너스 공식 웹사이트 제작 용역

## 1. IA 원칙

- 클라이언트는 "지금 상태", "내가 할 일", "승인할 것"이 첫 화면에서 보여야 한다.
- 내부 사용자는 "전체 프로젝트 통제", "지연 감지", "범위 통제", "승인 관리"가 우선이다.
- 모든 화면은 `내부 전용`과 `클라이언트 공유` 데이터가 명확히 분리되어야 한다.
- 프로젝트 상세는 `Overview` 중심 구조로 설계하고, 나머지 화면은 세부 탭으로 나눈다.
- 빈파트너스처럼 단일 웹사이트 제작 프로젝트부터 시작하지만 다중 프로젝트 고객사도 수용 가능해야 한다.

## 4. 공통 레이아웃

### 4.1 데스크톱

- 좌측 사이드바: 1차 메뉴
- 상단 바: 현재 조직/프로젝트, 검색, 알림, 프로필
- 메인 영역: 페이지 본문
- 우측 보조 패널: 승인 대기, 마감 임박, 최근 활동

### 4.2 모바일

- 하단 탭바: 대시보드, 프로젝트, 메시지, 승인, 계정
- 프로젝트 화면은 탭 대신 드롭다운 섹션 전환
- 관리자 화면은 모바일 최소 지원, 실무 운영은 태블릿 이상 권장

## 5. 클라이언트 화면 상세

### 5.1 `/login`

목적:

- 초대받은 클라이언트가 안전하게 로그인

주요 요소:

- 이메일
- 비밀번호
- 비밀번호 재설정
- 브랜드 소개 짧은 문구

### 5.2 `/dashboard`

목적:

- 로그인 후 현재 진행 상황을 10초 안에 파악

핵심 카드:

- 진행 중 프로젝트
- 현재 단계
- 전체 진척률
- 다음 액션
- 승인 대기
- 자료 제출 요청
- 최근 업데이트
- 새 메시지
- 다가오는 미팅

빈파트너스 예시 첫 화면:

- 현재 단계: `자료 수집`
- 자료 제출 마감: `2026-03-18`
- 다음 공식 협의: `디자인 컨셉 공유`
- 승인 대기: `없음` 또는 `메인 구조 승인`

### 5.3 `/projects/[projectId]/overview`

목적:

- 프로젝트 전체 상태를 한 화면에서 요약

섹션:

- 프로젝트 헤더
- 계약 정보 카드
- 현재 단계 카드
- 다음 액션 카드
- 최근 공식 업데이트
- 주요 일정 타임라인
- 승인 대기 요약
- 서드파티 서비스 상태

주요 CTA:

- 자료 제출하기
- 메시지 남기기
- 승인 보기

### 5.4 `/projects/[projectId]/timeline`

목적:

- 계약부터 하자보수까지 일정을 단계별로 공유

섹션:

- 마일스톤 세로 타임라인
- 각 단계 상태 배지
- 단계별 시작일/마감일
- 지연 사유 표시
- 관련 파일/업데이트/승인 링크

### 5.5 `/projects/[projectId]/tasks`

목적:

- 클라이언트가 해야 할 일과 내부 진행 상태를 구분해서 확인

섹션:

- 클라이언트 액션 필요 목록
- 진행 중 작업 목록
- 완료 작업 목록
- 필터: 전체, 내 액션, 승인 대기, 완료

노출 규칙:

- 내부 전용 태스크는 클라이언트에게 노출하지 않음

### 5.6 `/projects/[projectId]/updates`

목적:

- 공식 진행 공유 로그 확인

섹션:

- 업데이트 리스트
- 진행률 변화
- 클라이언트 액션 필요 배지
- 첨부 파일

정렬:

- 최신순

### 5.7 `/projects/[projectId]/files`

목적:

- 계약서, 시안, 전달 파일, 정책 문서를 한 곳에서 확인

필터:

- 계약
- 자료
- 디자인
- 개발
- 납품
- 미팅

주요 동작:

- 파일 미리보기
- 버전 라벨 확인
- 다운로드

### 5.8 `/projects/[projectId]/approvals`

목적:

- 디자인 승인, 기능 승인, 검수 승인 상태를 명확히 관리

섹션:

- 승인 대기 목록
- 승인 완료 이력
- 반려 이력
- 자동 승인 예정 항목

주요 정보:

- 요청일
- 응답 마감일
- 요청 사유
- 첨부 파일/버전
- 승인자

### 5.9 `/projects/[projectId]/messages`

목적:

- 프로젝트 커뮤니케이션을 카톡 대신 포털 내부에 정리

섹션:

- 대화 목록
- 현재 스레드
- 첨부 파일
- 읽음 상태

규칙:

- 내부 메모는 클라이언트에게 보이지 않음

### 5.10 `/projects/[projectId]/meetings`

목적:

- 공식 협의 3회를 포함한 회의 이력 관리

섹션:

- 예정된 미팅
- 지난 미팅
- 미팅록
- 결정사항
- 후속 액션

빈파트너스 기본 생성 항목:

- 디자인 컨셉 공유
- 기능 협의
- 최종 완료 및 오픈 점검

### 5.11 `/projects/[projectId]/change-requests`

목적:

- 추가 수정 요청과 유지보수 범위를 분리

섹션:

- 요청 목록
- 범위 포함 여부
- 일정 영향
- 비용 영향
- 승인 상태

주요 메시지:

- 계약 범위 포함
- 별도 유지보수
- 신규 범위

### 5.12 `/projects/[projectId]/handover`

목적:

- 오픈 이후 운영 전환 상태를 확인

섹션:

- 도메인 인수
- Framer 호스팅 전환일
- AI 챗봇 비용 전환일
- 뉴스레터 운영 주체
- 예약 캘린더 운영 주체
- 교육 완료 여부
- 하자보수 종료일

## 6. 관리자 화면 상세

### 6.1 `/admin/overview`

목적:

- 전체 프로젝트 상황을 한 화면에서 통제

핵심 위젯:

- 진행 중 프로젝트 수
- 승인 대기 수
- 지연 프로젝트 수
- 자료 미제출 고객 수

- 최근 활동 로그

목적:

- 상태
- 담당자
- 고객사

- 지연

테이블 컬럼:

- 프로젝트명
- 고객사
- 현재 단계
- 진척률
- 종료 예정일
- 다음 액션
- 담당자

목적:

- 담당자 지정

## 8. 빈파트너스 초기 화면 상태 예시

### 8.1 클라이언트 대시보드

- 프로젝트명: 빈파트너스 공식 웹사이트 제작
- 상태: `collecting_content`
- 전체 진척률: `15%`
- 자료 제출 마감: `2026-03-18`
- 다음 미팅: `디자인 컨셉 공유`
- 승인 대기: `0건`

### 8.2 관리자 프로젝트 개요

- 자료 요청 8건 중 3건 제출
- 공식 업데이트 1건 발행
- 다음 액션: 메인 시안 업로드
- 리스크: 서비스 상세 문안 미수급

## 10. 결론

이 IA의 핵심은 "클라이언트는 단순하고 명확하게, 내부는 통제 가능하게"다.

빈파트너스 같은 첫 실사용 프로젝트에서는 아래 3개 화면이 특히 중요하다.

- 클라이언트 대시보드
- 관리자 프로젝트 개요
- 승인 화면

이 3개 화면 품질이 좋으면 실제 운영 전환 속도가 가장 빨라진다.
